# 2. プロジェクト構造

このプロジェクトのフォルダやファイルが、それぞれどのような意味を持っているのかを知ることで、コードのどこに何が書かれているのかを把握しやすくなります。

ルートディレクトリ（一番上の階層）にあるファイルと、特に重要な `src` フォルダの中身を見ていきましょう。

## ルートディレクトリの主なファイル

```
codename/
├── .gitignore          # Git（バージョン管理）で無視するファイルを指定
├── package.json        # プロジェクトの情報や依存ライブラリを管理
├── vite.config.ts      # Vite（ビルドツール）の設定ファイル
├── tsconfig.json       # TypeScriptの設定ファイル
├── index.html          # Webページの土台となるHTMLファイル
├── src/                # ★プログラムの本体となるソースコードが入っている最重要フォルダ
└── study_files/        # この解説書が置かれているフォルダ
```

*   **`package.json`**:
    このプロジェクトがどんなライブラリ（`dependencies`）に依存しているか、どんなコマンド（`scripts`）が使えるかが書かれています。例えば、`npm run dev` コマンドで開発サーバーを起動できるのは、このファイルに `"dev": "vite"` と書かれているおかげです。

*   **`index.html`**:
    すべての表示内容の基礎となるファイルです。`<body>` タグの中に `<div id="root"></div>` という記述があります。Reactはこの `div` タグの中に、コンポーネントを組み合わせて作った画面を描画していきます。

*   **`src` フォルダ**:
    "source"（ソースコード）の略で、アプリケーションの心臓部です。私たちが書くプログラムのほとんどは、このフォルダの中に置かれます。

## `src` フォルダの中身

```
src/
├── main.tsx            # プログラムの出発点
├── App.tsx             # アプリケーション全体の親玉コンポーネント
├── gameLogic.ts        # ゲームのルールなど、中心的なロジック
├── types.ts            # データ型を定義するファイル
├── index.css           # 全体に適応されるスタイル
├── components/         # 再利用可能なUI部品（コンポーネント）置き場
│   ├── Board.tsx       # ゲーム盤
│   └── Card.tsx        # カード1枚
├── data/               # ゲームで使うデータ置き場
│   └── words.json      # コードネームの単語リスト
└── assets/             # 画像ファイルなど
```

*   **`main.tsx`**:
    Reactアプリケーションを起動し、`index.html` の `<div id="root"></div>` に `App` コンポーネントを埋め込む役割を持つ、プログラムの「玄関」です。

*   **`App.tsx`**:
    ヘッダー、ゲーム盤、情報表示エリアなど、このアプリケーションのすべての要素をまとめている「親」となるコンポーネントです。ゲーム全体の「状態（`gameState`）」を管理する重要な役割も担っています。

*   **`gameLogic.ts`**:
    「新しいゲームを開始する」「カードを選ぶ」「ターンをパスする」といった、**見た目（UI）とは直接関係のない、ゲームのルールそのもの**が書かれています。UIを担当する `App.tsx` から呼び出される形で使われます。このようにロジックを分離することで、コードの見通しが良くなります。

*   **`types.ts`**:
    `gameLogic.ts` や `App.tsx` で使われるデータの「型」を定義しています。例えば、「ゲームの状態 `GameState` とは、ゲーム盤 `board` や現在のチーム `currentTeam` を持つものである」といったルールを定めています。

*   **`components` フォルダ**:
    UIを構成する部品が置かれています。
    *   `Card.tsx` はカード1枚の表示を担当します。
    *   `Board.tsx` は `Card.tsx` を25個並べてゲーム盤を形成します。
    このように部品を分けることで、修正や再利用がしやすくなります。

*   **`data` フォルダ**:
    ゲームで使う単語リストのような、プログラムのコードではない静的なデータが置かれています。

---

このように、ファイルやフォルダが役割ごとに整理されているのが分かります。
何かを探したいとき、例えば「カードのデザインを変えたい」なら `Card.tsx` や関連するCSSを、「単語を追加したい」なら `words.json` を見ればよい、という見当がつけられるようになります。
