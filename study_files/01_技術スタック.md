# 1. 技術スタック

このプロジェクトは、現代的なWebアプリケーションで広く使われている技術を組み合わせて作られています。
それぞれの技術がどのような役割を担っているのかを見ていきましょう。

## 主要な技術

| 技術名 | 種類 | 役割 |
| :--- | :--- | :--- |
| **TypeScript** | プログラミング言語 | JavaScriptを拡張して、より安全で大規模な開発にも対応できるようにしたもの。 |
| **React** | ライブラリ | Facebookが開発した、UI（ユーザーインターフェース）を効率的に作るためのもの。 |
| **Vite** | ビルドツール | 開発中は高速な動作を、公開時には最適化されたファイルを提供してくれる縁の下の力持ち。 |
| **CSS** | スタイルシート言語| 文字の色や背景、レイアウトなど、Webページの見た目を整えるためのもの。 |

## なぜこれらの技術が使われているのか？

一言で言えば、「**モダンで効率的な開発をするため**」です。

### TypeScript: 安全性と開発しやすさ

JavaScriptは非常に柔軟な言語ですが、その柔軟さが原因で、意図しないエラー（バグ）を生みやすい側面もあります。
TypeScriptは、「型」というルールをコードに加えることで、開発の早い段階でエラーを見つけやすくしてくれます。

**具体例 (`src/types.ts`)**:

このファイルでは、ゲームで使われるデータ（カードの種類、チーム、ゲームの状態など）の「形」を厳密に定義しています。

```typescript
export type CardRole = "red" | "blue" | "neutral" | "assassin";
export type Team = "red" | "blue";

export type CardType = {
  word: string;
  role: CardRole;
  revealed: boolean;
};
```

このように定義することで、例えば `CardType` の `role` に `"green"` のような想定外の値が入ることを防ぎ、プログラムが予期せぬ挙動をするのを未然に防ぎます。

### React: UI構築の効率化

Reactは「コンポーネント」という部品単位でUIを管理します。例えば、このゲームでは「カード1枚」や「ゲーム盤全体」がそれぞれコンポーネントとして作られています。

*   **`src/components/Card.tsx`**: カード1枚の見た目と振る舞いを定義しています。
*   **`src/components/Board.tsx`**: `Card.tsx` コンポーネントを25枚並べて、ゲーム盤全体を構成しています。

このように部品を組み合わせることで、複雑な画面でも整理されたコードで効率的に開発を進めることができます。
また、ゲームの状態（例：赤チームの残り枚数）が変化したときに、**画面のどの部分を再描画すればよいかをReactが自動で判断してくれる**ため、開発者は表示の更新処理を細かく気にする必要がありません。

### Vite: 快適な開発体験

従来の開発ツールでは、コードを少し変更しただけでも、ブラウザで確認できるまでに数秒〜数十秒待たされることがありました。
Viteは、非常に高速に動作する開発サーバーを提供してくれるため、コードの変更がほぼ瞬時にブラウザに反映されます。これにより、試行錯誤のサイクルを速く回すことができ、開発のストレスが大幅に軽減されます。

`vite.config.ts` という設定ファイルで、Viteの挙動をコントロールしています。

---

これらの技術は、単体でも強力ですが、組み合わせることで、開発しやすく、メンテナンスしやすく、そして高速に動作するWebアプリケーションを作ることができます。
