# プロジェクト引き継ぎ資料: コードネーム風ゲーム

## 1. プロジェクト概要
React + TypeScript + Vite で作成された、ブラウザで遊べる「コードネーム」風のボードゲームです。
2つのチーム（赤・青）に分かれ、スパイマスターのヒントを元に、エージェントが自チームの単語カードを当てるゲームです。

## 2. 技術スタック
- **Framework**: React (Vite)
- **Language**: TypeScript
- **Styling**: CSS (`App.css`, `GameInfo.css`)
- **State Management**: React `useState` + `localStorage` (ブラウザリロード時の状態保持用)

## 3. ディレクトリ構成と主要ファイルの役割
現在のソースコードの状況は以下の通りです。

- **`src/App.tsx`**:
    - アプリケーションのメインコンポーネント。
    - **役割**: ゲーム全体の状態管理 (`gameState`)、UIの構築、イベントハンドリング（カードクリック、リセット、表示切替）。
    - **特徴**: `localStorage` を利用してゲーム状態を永続化しています。
- **`src/gameLogic.ts`**:
    - ゲームのコアルロジック（純粋関数）。
    - **役割**:
        - `createNewGame()`: 初期盤面の生成、単語のシャッフル、役割（赤/青/中立/アサシン）の割り当て。
        - `selectCard()`: カード選択時の判定、ターン交代、勝敗判定のロジック。
- **`src/types.ts`**:
    - 型定義ファイル。
    - `GameState`, `CardType`, `Team`, `CardRole` などを定義。
- **`src/components/Board.tsx`** (想定):
    - 盤面描画コンポーネント。`gameState.board` を受け取りグリッド表示します。
- **`src/components/Card.tsx`** (想定):
    - カード単体のコンポーネント。単語表示、クリックイベント、スタイリング（めくられた状態など）を担当。
- **`src/data/words.json`** (想定):
    - ゲームに使用する単語リスト。

## 5. 次に取り組むべきタスク: 「パス」機能の実装
**目的**: 自分のターンに単語を選ばず、相手のターンに交代する機能を追加する。

**要件**:
1.  **UI**: 画面上部（`App.tsx` の `controls` エリアなど）に「パス（ターン終了）」ボタンを追加する。
2.  **挙動**: ボタンを押すと、カードをめくらずに即座に相手チームのターンになる。
3.  **制限**: パス回数に制限は設けない。
